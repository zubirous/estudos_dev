# Prova: VIEWS - MySQL

## Provas de Verdadeiro ou Falso

## Questão 1 (Verdadeiro/Falso)
VIEWs armazenam dados físicos no banco de dados.

**Resposta:** Falso. VIEWs são consultas SQL armazenadas, não armazenam dados físicos. Os dados vêm das tabelas base.

---

## Questão 2 (Verdadeiro/Falso)
VIEWs podem ser usadas como se fossem tabelas em consultas SELECT.

**Resposta:** Verdadeiro. VIEWs podem ser consultadas como tabelas normais usando SELECT.

---

## Questão 3 (Verdadeiro/Falso)
VIEWs são úteis para simplificar consultas complexas.

**Resposta:** Verdadeiro. VIEWs são especialmente úteis para simplificar e reutilizar consultas complexas.

---

## Questão 4 (Verdadeiro/Falso)
É possível inserir dados diretamente em uma VIEW.

**Resposta:** Falso (geralmente). VIEWs simples podem permitir INSERT/UPDATE em alguns casos, mas nem todas VIEWs suportam isso. VIEWs são principalmente para leitura.

---

## Questão 5 (Verdadeiro/Falso)
VIEWs podem ser atualizadas usando CREATE OR REPLACE VIEW.

**Resposta:** Verdadeiro. CREATE OR REPLACE VIEW permite atualizar a definição de uma VIEW existente.

---

## Questão 6 (Verdadeiro/Falso)
VIEWs melhoram a segurança ao ocultar colunas sensíveis.

**Resposta:** Verdadeiro. VIEWs podem selecionar apenas colunas necessárias, ocultando dados sensíveis dos usuários.

---

## Questão 7 (Verdadeiro/Falso)
VIEWs podem conter JOINs e subconsultas.

**Resposta:** Verdadeiro. VIEWs são consultas SQL completas, podendo incluir JOINs, subconsultas, agregações, etc.
