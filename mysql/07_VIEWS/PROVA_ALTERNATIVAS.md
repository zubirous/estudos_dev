# Prova: VIEWS - MySQL

## Provas de Alternativas

## Questão 1 (Múltipla Escolha)
O que é uma VIEW no MySQL?

a) Uma tabela física
b) Uma consulta SQL armazenada como se fosse tabela
c) Um tipo de índice
d) Uma constraint

**Resposta:** b) Uma consulta SQL armazenada como se fosse tabela

---

## Questão 2 (Múltipla Escolha)
Qual comando cria uma VIEW?

a) CREATE TABLE
b) CREATE VIEW
c) CREATE INDEX
d) CREATE QUERY

**Resposta:** b) CREATE VIEW

---

## Questão 3 (Múltipla Escolha)
Para que são úteis as VIEWs?

a) Armazenar dados físicos
b) Simplificar consultas complexas e reutilizar lógica
c) Criar índices
d) Definir constraints

**Resposta:** b) Simplificar consultas complexas e reutilizar lógica

---

## Questão 4 (Múltipla Escolha)
Como atualizar uma VIEW existente?

a) ALTER VIEW
b) UPDATE VIEW
c) CREATE OR REPLACE VIEW
d) MODIFY VIEW

**Resposta:** c) CREATE OR REPLACE VIEW

---

## Questão 5 (Múltipla Escolha)
Como deletar uma VIEW?

a) DELETE VIEW
b) DROP VIEW
c) REMOVE VIEW
d) DROP TABLE

**Resposta:** b) DROP VIEW

---

## Questão 6 (Múltipla Escolha)
VIEWs são especialmente úteis para:

a) Armazenar grandes volumes de dados
b) Simplificar consultas complexas e melhorar segurança
c) Criar índices
d) Definir constraints

**Resposta:** b) Simplificar consultas complexas e melhorar segurança - ocultar dados sensíveis e simplificar queries

---

## Questão 7 (Múltipla Escolha)
O que acontece quando você consulta uma VIEW?

a) Os dados são armazenados na VIEW
b) A consulta SQL da VIEW é executada na hora
c) A VIEW retorna dados em cache
d) Nada acontece

**Resposta:** b) A consulta SQL da VIEW é executada na hora - VIEWs não armazenam dados, executam a query quando consultadas
