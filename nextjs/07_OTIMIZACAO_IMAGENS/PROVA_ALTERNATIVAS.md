# Prova: OTIMIZAÇÃO DE IMAGENS - Next.js

## Provas de Alternativas

## Questão 1 (Múltipla Escolha)
Qual componente do Next.js deve ser usado para imagens?

a) <img>
b) <Image> do next/image
c) <Picture>
d) Qualquer um

**Resposta:** b) <Image> do next/image

---

## Questão 2 (Múltipla Escolha)
Quais são os benefícios do componente Image do Next.js?

a) Otimização automática, lazy loading, formato WebP
b) Apenas otimização
c) Apenas lazy loading
d) Não tem benefícios

**Resposta:** a) Otimização automática, lazy loading, formato WebP quando suportado

---

## Questão 3 (Múltipla Escolha)
Como configurar imagens de domínios externos no Next.js?

a) Não é possível
b) Usar tag <img>
c) Configurar em next.config.js no campo images.domains
d) Não precisa configurar

**Resposta:** c) Configurar em next.config.js no campo images.domains

---

## Questão 4 (Múltipla Escolha)
O componente Image do Next.js faz:

a) Apenas otimização
b) Otimização automática, lazy loading, conversão de formato
c) Apenas lazy loading
d) Nada

**Resposta:** b) Otimização automática, lazy loading, conversão de formato

---

## Questão 5 (Múltipla Escolha)
Para usar imagens externas no componente Image, é necessário:

a) Nada, funciona automaticamente
b) Configurar o domínio em next.config.js
c) Usar tag <img>
d) Não é possível usar imagens externas

**Resposta:** b) Configurar o domínio em next.config.js

---

## Questão 6 (Múltipla Escolha)
O que é lazy loading de imagens?

a) Carregar todas as imagens de uma vez
b) Carregar imagens apenas quando estão prestes a aparecer na tela
c) Não carregar imagens
d) Carregar imagens apenas no servidor

**Resposta:** b) Carregar imagens apenas quando estão prestes a aparecer na tela

---

## Questão 7 (Múltipla Escolha)
O componente Image do Next.js converte imagens para:

a) PNG
b) JPEG
c) WebP quando suportado
d) GIF

**Resposta:** c) WebP quando suportado pelo navegador

---

## Questão 8 (Múltipla Escolha)
Por que usar Image ao invés de <img> no Next.js?

a) Não há diferença
b) Image tem otimizações automáticas que <img> não tem
c) <img> é melhor
d) Image não funciona

**Resposta:** b) Image tem otimizações automáticas que <img> não tem

---

## Questão 9 (Múltipla Escolha)
O componente Image do Next.js é obrigatório?

a) Sim, sempre
b) Não, mas recomendado para melhor performance
c) Nunca usar
d) Apenas para imagens externas

**Resposta:** b) Não, mas recomendado para melhor performance

---

## Questão 10 (Múltipla Escolha)
O componente Image requer quais propriedades obrigatórias?

a) Apenas src
b) src, alt, width, height
c) Apenas alt
d) Nenhuma

**Resposta:** b) src, alt, width, height - todas são obrigatórias
