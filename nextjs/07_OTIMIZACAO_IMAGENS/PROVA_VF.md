# Prova: OTIMIZAÇÃO DE IMAGENS - Next.js

## Provas de Verdadeiro ou Falso

## Questão 1 (Verdadeiro/Falso)
O componente Image do Next.js faz otimização automática.

**Resposta:** Verdadeiro. O componente Image faz otimização automática de imagens, incluindo redimensionamento e conversão de formato.

---

## Questão 2 (Verdadeiro/Falso)
O componente Image do Next.js faz lazy loading automaticamente.

**Resposta:** Verdadeiro. O componente Image carrega imagens apenas quando estão prestes a aparecer na tela (lazy loading automático).

---

## Questão 3 (Verdadeiro/Falso)
É possível usar imagens externas no componente Image sem configuração.

**Resposta:** Falso. É necessário configurar o domínio em next.config.js antes de usar imagens externas.

---

## Questão 4 (Verdadeiro/Falso)
O componente Image converte imagens para WebP automaticamente.

**Resposta:** Verdadeiro. O componente Image converte imagens para WebP quando o navegador suporta, melhorando performance.

---

## Questão 5 (Verdadeiro/Falso)
O componente Image requer width e height para imagens estáticas.

**Resposta:** Verdadeiro. Para imagens estáticas, width e height são obrigatórios para evitar layout shift.

---

## Questão 6 (Verdadeiro/Falso)
O componente Image funciona apenas com imagens locais.

**Resposta:** Falso. O componente Image funciona com imagens locais e externas (após configurar domínios).

---

## Questão 7 (Verdadeiro/Falso)
O componente Image melhora a performance da página.

**Resposta:** Verdadeiro. O componente Image melhora performance através de otimização automática, lazy loading e formatos modernos como WebP.

---

## Questão 8 (Verdadeiro/Falso)
A tag <img> tem as mesmas otimizações que o componente Image.

**Resposta:** Falso. A tag <img> não tem otimizações automáticas. O componente Image tem otimização, lazy loading e conversão de formato.
