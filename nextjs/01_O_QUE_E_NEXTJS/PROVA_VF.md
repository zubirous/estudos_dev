# Prova: O QUE É NEXT.JS?

## Provas de Verdadeiro ou Falso

## Questão 1 (Verdadeiro/Falso)
Next.js é um framework React para produção.

**Resposta:** Verdadeiro. Next.js é um framework React criado pela Vercel para desenvolvimento de aplicações React otimizadas para produção.

---

## Questão 2 (Verdadeiro/Falso)
Next.js suporta Server-Side Rendering (SSR).

**Resposta:** Verdadeiro. Next.js suporta SSR através de getServerSideProps.

---

## Questão 3 (Verdadeiro/Falso)
Next.js usa roteamento baseado em arquivos.

**Resposta:** Verdadeiro. Next.js usa file-based routing, onde cada arquivo em pages/ vira uma rota automaticamente.

---

## Questão 4 (Verdadeiro/Falso)
Next.js pode criar APIs dentro do próprio projeto.

**Resposta:** Verdadeiro. Next.js permite criar API Routes em pages/api/.

---

## Questão 5 (Verdadeiro/Falso)
Next.js faz code splitting automático.

**Resposta:** Verdadeiro. Next.js faz automaticamente code splitting, carregando apenas o código necessário para cada página.

---

## Questão 6 (Verdadeiro/Falso)
Next.js pode criar APIs dentro do mesmo projeto.

**Resposta:** Verdadeiro. Next.js permite criar API Routes em pages/api/, eliminando necessidade de servidor separado.

---

## Questão 7 (Verdadeiro/Falso)
Next.js é melhor que React puro apenas para SEO.

**Resposta:** Falso. Next.js é melhor para SEO, SSR, SSG, otimizações automáticas, e desenvolvimento mais rápido, não apenas SEO.

---

## Questão 8 (Verdadeiro/Falso)
Next.js otimiza imagens automaticamente.

**Resposta:** Verdadeiro. Next.js otimiza imagens automaticamente através do componente Image, convertendo para WebP e fazendo lazy loading.

---

## Questão 9 (Verdadeiro/Falso)
Next.js suporta apenas Server-Side Rendering.

**Resposta:** Falso. Next.js suporta CSR (Client-Side Rendering), SSR (Server-Side Rendering) e SSG (Static Site Generation).
