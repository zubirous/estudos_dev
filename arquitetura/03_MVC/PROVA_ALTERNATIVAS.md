# Prova: MVC (Model-View-Controller)

## Provas de Alternativas

## Questão 1 (Múltipla Escolha)
Quais são os três componentes do MVC?

a) Model, View, Controller
b) Model, View, Presenter
c) Model, View, ViewModel
d) Manager, View, Controller

**Resposta:** a) Model, View, Controller

---

## Questão 2 (Múltipla Escolha)
No MVC, quem é responsável por receber a entrada do usuário?

a) Model
b) View
c) Controller
d) Service

**Resposta:** c) Controller

---

## Questão 3 (Múltipla Escolha)
Qual componente do MVC não deve acessar diretamente outro componente?

a) Controller pode acessar Model e View
b) View não deve acessar Model diretamente
c) Model pode acessar View
d) Controller não pode acessar Model

**Resposta:** b) View não deve acessar Model diretamente no MVC clássico.

---

## Questão 4 (Múltipla Escolha)
No MVC, o Controller:

a) Contém lógica de negócio
b) Coordena Model e View
c) Exibe dados
d) Armazena dados

**Resposta:** b) Coordena Model e View - Controller é o intermediário

---

## Questão 5 (Múltipla Escolha)
No MVC, o Model:

a) Interface do usuário
b) Dados e lógica de negócio
c) Coordenação
d) Nada

**Resposta:** b) Dados e lógica de negócio - Model gerencia dados

---

## Questão 6 (Múltipla Escolha)
No MVC, o fluxo típico é:

a) Usuário → View → Model → Controller
b) Usuário → Controller → Model → Controller → View → Usuário
c) Usuário → Model → View
d) Não há fluxo

**Resposta:** b) Usuário → Controller → Model → Controller → View → Usuário

---

## Questão 7 (Múltipla Escolha)
Qual é uma vantagem do MVC?

a) Separação clara de responsabilidades
b) Fácil manutenção
c) Reutilização de Models e Views
d) Todas as alternativas acima

**Resposta:** d) Todas as alternativas acima

---

## Questão 8 (Múltipla Escolha)
O problema do "Fat Controller" ocorre quando:

a) Controller fica com muita lógica de negócio
b) Controller é muito pequeno
c) Controller não faz nada
d) Não há problema

**Resposta:** a) Controller fica com muita lógica de negócio - lógica deve estar no Model ou Services

---

## Questão 9 (Múltipla Escolha)
No MVC clássico, quem recebe input do usuário?

a) Model
b) View
c) Controller
d) Service

**Resposta:** c) Controller - recebe input e decide ação

---

## Questão 10 (Múltipla Escolha)
No MVC, quem é responsável por exibir dados ao usuário?

a) Model
b) View
c) Controller
d) Service

**Resposta:** b) View - interface do usuário que exibe dados
