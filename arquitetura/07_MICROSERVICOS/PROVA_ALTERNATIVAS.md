# Prova: ARQUITETURA MICROSERVIÇOS

## Provas de Alternativas

## Questão 1 (Múltipla Escolha)
Qual característica define microserviços?

a) Um único banco de dados compartilhado
b) Cada serviço tem seu próprio banco de dados
c) Apenas um serviço no sistema
d) Serviços sempre usam a mesma tecnologia

**Resposta:** b) Cada serviço tem seu próprio banco de dados

---

## Questão 2 (Múltipla Escolha)
Quando escolher arquitetura de microserviços?

a) Sistema pequeno e simples
b) Sistema grande com necessidade de escalar partes específicas
c) Aplicação de demonstração
d) Sempre escolher microserviços

**Resposta:** b) Sistema grande com necessidade de escalar partes específicas

---

## Questão 3 (Múltipla Escolha)
Como os microserviços se comunicam?

a) Via banco de dados compartilhado
b) Via APIs (REST, gRPC, etc)
c) Via arquivos compartilhados
d) Não se comunicam

**Resposta:** b) Via APIs (REST, gRPC, etc) - comunicação via protocolos de rede

---

## Questão 4 (Múltipla Escolha)
Qual é uma vantagem dos microserviços?

a) Escalabilidade independente
b) Deploy independente
c) Tecnologias diferentes por serviço
d) Todas as alternativas acima

**Resposta:** d) Todas as alternativas acima

---

## Questão 5 (Múltipla Escolha)
Qual é uma desvantagem dos microserviços?

a) Complexidade de comunicação
b) Consistência distribuída é difícil
c) Maior overhead operacional
d) Todas as alternativas acima

**Resposta:** d) Todas as alternativas acima

---

## Questão 6 (Múltipla Escolha)
Em microserviços, falhas são:

a) Isoladas por serviço
b) Afetam todo sistema
c) Sempre críticas
d) Nunca ocorrem

**Resposta:** a) Isoladas por serviço - uma falha em um serviço não derruba todo sistema
