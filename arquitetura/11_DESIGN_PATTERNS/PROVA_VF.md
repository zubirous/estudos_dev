# Prova: DESIGN PATTERNS

## Provas de Verdadeiro ou Falso

## Questão 1 (Verdadeiro/Falso)
Singleton garante apenas uma instância de uma classe.

**Resposta:** Verdadeiro. Singleton é um padrão que garante que uma classe tenha apenas uma instância.

---

## Questão 2 (Verdadeiro/Falso)
Factory é usado para criar objetos sem especificar a classe exata.

**Resposta:** Verdadeiro. Factory pattern permite criar objetos através de uma factory sem especificar a classe exata.

---

## Questão 3 (Verdadeiro/Falso)
Repository abstrai acesso a dados.

**Resposta:** Verdadeiro. Repository pattern abstrai acesso a dados, permitindo trocar implementação (MySQL, MongoDB, etc) sem alterar código de negócio.

---

## Questão 4 (Verdadeiro/Falso)
Observer define sistema de notificações um-para-muitos.

**Resposta:** Verdadeiro. Observer pattern define dependência um-para-muitos, onde um objeto notifica múltiplos observadores.

---

## Questão 5 (Verdadeiro/Falso)
Strategy permite trocar algoritmos em tempo de execução.

**Resposta:** Verdadeiro. Strategy pattern define família de algoritmos intercambiáveis que podem ser trocados em tempo de execução.

---

## Questão 6 (Verdadeiro/Falso)
Design Patterns são soluções reutilizáveis para problemas comuns de design.

**Resposta:** Verdadeiro. Design Patterns são soluções reutilizáveis para problemas recorrentes de design de software.

---

## Questão 7 (Verdadeiro/Falso)
O padrão Factory permite criar objetos sem especificar a classe exata.

**Resposta:** Verdadeiro. Factory pattern permite criar objetos através de uma factory sem especificar a classe exata do objeto criado.

---

## Questão 8 (Verdadeiro/Falso)
Design Patterns devem ser usados sempre, independentemente da situação.

**Resposta:** Falso. Design Patterns devem ser usados quando resolvem um problema específico. Usar patterns desnecessariamente adiciona complexidade.

---

## Questão 9 (Verdadeiro/Falso)
O padrão Strategy permite definir família de algoritmos intercambiáveis.

**Resposta:** Verdadeiro. Strategy pattern define família de algoritmos que podem ser trocados em tempo de execução.
