# Prova: DESIGN PATTERNS

## Provas de Alternativas

## Questão 1 (Múltipla Escolha)
Qual padrão garante apenas uma instância de uma classe?

a) Factory
b) Singleton
c) Repository
d) Observer

**Resposta:** b) Singleton

---

## Questão 2 (Múltipla Escolha)
Qual padrão é usado para criar objetos sem especificar a classe exata?

a) Factory
b) Singleton
c) Repository
d) Strategy

**Resposta:** a) Factory

---

## Questão 3 (Múltipla Escolha)
Qual padrão abstrai acesso a dados?

a) Factory
b) Singleton
c) Repository
d) Observer

**Resposta:** c) Repository - abstrai acesso a dados, permitindo trocar implementação

---

## Questão 4 (Múltipla Escolha)
Qual padrão define dependência um-para-muitos entre objetos?

a) Factory
b) Singleton
c) Repository
d) Observer

**Resposta:** d) Observer - define sistema de notificações um-para-muitos

---

## Questão 5 (Múltipla Escolha)
Qual padrão define família de algoritmos intercambiáveis?

a) Factory
b) Singleton
c) Repository
d) Strategy

**Resposta:** d) Strategy - permite trocar algoritmos em tempo de execução

---

## Questão 6 (Múltipla Escolha)
O padrão Repository é útil para:

a) Criar objetos
b) Abstrair acesso a dados e trocar implementação (MySQL, MongoDB, etc)
c) Garantir instância única
d) Notificar objetos

**Resposta:** b) Abstrair acesso a dados e trocar implementação (MySQL, MongoDB, etc)

---

## Questão 7 (Múltipla Escolha)
Qual padrão permite trocar algoritmos em tempo de execução?

a) Factory
b) Singleton
c) Strategy
d) Observer

**Resposta:** c) Strategy - define família de algoritmos intercambiáveis

---

## Questão 8 (Múltipla Escolha)
Design Patterns são:

a) Soluções reutilizáveis para problemas comuns de design
b) Arquiteturas de sistema
c) Linguagens de programação
d) Frameworks

**Resposta:** a) Soluções reutilizáveis para problemas comuns de design

---

## Questão 9 (Múltipla Escolha)
Qual padrão é usado quando precisa garantir que uma classe tenha apenas uma instância?

a) Factory
b) Singleton
c) Repository
d) Strategy

**Resposta:** b) Singleton - garante apenas uma instância de uma classe

---

## Questão 10 (Múltipla Escolha)
O padrão Observer é útil para:

a) Criar objetos
b) Sistema de notificações onde um objeto notifica múltiplos observadores
c) Garantir instância única
d) Abstrair acesso a dados

**Resposta:** b) Sistema de notificações onde um objeto notifica múltiplos observadores
