# Prova: ARQUITETURA vs PADRÃO DE PROJETO

## Provas de Alternativas

## Questão 1 (Múltipla Escolha)
O que é **arquitetura de software**?

a) Um padrão de código para classes específicas  
b) Estrutura geral do sistema, organizando componentes de alto nível  
c) Um conjunto de testes automatizados  
d) Um padrão de projeto focado em botões da interface  

**Resposta:** b) Estrutura geral do sistema, organizando componentes de alto nível

---

## Questão 2 (Múltipla Escolha)
O que é um **padrão de projeto (Design Pattern)**?

a) Um banco de dados específico usado pelo sistema  
b) Uma solução reutilizável para problemas específicos de design  
c) Um tipo de arquitetura cliente-servidor  
d) Um framework de interface gráfica  

**Resposta:** b) Uma solução reutilizável para problemas específicos de design

---

## Questão 3 (Múltipla Escolha)
Qual a diferença principal entre **arquitetura** e **padrão de projeto**?

a) Arquitetura é código, padrão é documentação  
b) Arquitetura resolve detalhes, padrões resolvem o sistema inteiro  
c) Arquitetura trata do sistema completo; padrões tratam de problemas específicos  
d) Não há diferença, são sinônimos  

**Resposta:** c) Arquitetura trata do sistema completo; padrões tratam de problemas específicos

---

## Questão 4 (Múltipla Escolha)
Qual dos itens abaixo é um **exemplo de arquitetura**?

a) Singleton  
b) Factory Method  
c) Observer  
d) MVC  

**Resposta:** d) MVC

---

## Questão 5 (Múltipla Escolha)
Na arquitetura **MVC**, o que representa o **Model**?

a) Interface do usuário (páginas HTML)  
b) Dados e lógica de negócio (ex.: Produto, Usuario)  
c) Coordenação entre componentes  
d) Regras de roteamento da aplicação  

**Resposta:** b) Dados e lógica de negócio (ex.: Produto, Usuario)

---

## Questão 6 (Múltipla Escolha)
Na arquitetura **MVC**, o que representa a **View**?

a) Banco de dados da aplicação  
b) Interface do usuário (páginas HTML)  
c) Camada de serviços externos  
d) Regras de negócio centrais  

**Resposta:** b) Interface do usuário (páginas HTML)

---

## Questão 7 (Múltipla Escolha)
Na arquitetura **MVC**, o que faz o **Controller**?

a) Armazena dados em disco  
b) Renderiza HTML diretamente no banco  
c) Coordena a interação entre Model e View  
d) Gera relatórios de log apenas  

**Resposta:** c) Coordena a interação entre Model e View

---

## Questão 8 (Múltipla Escolha)
Em relação ao momento de uso, qual afirmação está correta?

a) Arquitetura é decidida durante testes finais somente  
b) Padrões de projeto são definidos antes de qualquer decisão arquitetural  
c) Arquitetura é definida no início do projeto; padrões de projeto são aplicados durante a implementação  
d) Nem arquitetura nem padrões precisam ser planejados  

**Resposta:** c) Arquitetura é definida no início do projeto; padrões de projeto são aplicados durante a implementação
