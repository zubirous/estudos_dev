# Prova: MVP (Model-View-Presenter)

## Provas de Alternativas

## Questão 1 (Múltipla Escolha)
Qual a principal diferença entre MVC e MVP?

a) Não há diferença
b) No MVP, View é passiva e não conhece Model
c) No MVC, View é passiva
d) MVP tem ViewModel

**Resposta:** b) No MVP, View é passiva e não conhece Model

---

## Questão 2 (Múltipla Escolha)
No MVP, quem é responsável por atualizar a View?

a) Model
b) View
c) Presenter
d) Controller

**Resposta:** c) Presenter - ele atualiza a View diretamente

---

## Questão 3 (Múltipla Escolha)
No MVP, a View:

a) Conhece o Model diretamente
b) É passiva e apenas exibe dados
c) Contém lógica de negócio
d) Processa dados

**Resposta:** b) É passiva e apenas exibe dados

---

## Questão 4 (Múltipla Escolha)
Quem contém toda a lógica de apresentação no MVP?

a) Model
b) View
c) Presenter
d) Controller

**Resposta:** c) Presenter - contém toda lógica de apresentação

---

## Questão 5 (Múltipla Escolha)
No MVP, o Presenter:

a) Conhece View e Model
b) Só conhece View
c) Só conhece Model
d) Não conhece nem View nem Model

**Resposta:** a) Conhece View e Model - é o intermediário que coordena ambos

---

## Questão 6 (Múltipla Escolha)
Qual é a principal diferença entre MVP e MVC?

a) MVP tem Controller
b) MVP tem Presenter que é mais ativo na atualização da View
c) Não há diferença
d) MVP tem ViewModel

**Resposta:** b) MVP tem Presenter que é mais ativo na atualização da View

---

## Questão 7 (Múltipla Escolha)
No MVP, quem processa e formata dados para exibição?

a) Model
b) View
c) Presenter
d) Controller

**Resposta:** c) Presenter - toda lógica de apresentação está no Presenter

---

## Questão 8 (Múltipla Escolha)
Uma vantagem do MVP é:

a) View pode ser facilmente testada (é passiva)
b) Presenter pode ser testado independentemente
c) Separação clara de responsabilidades
d) Todas as alternativas acima

**Resposta:** d) Todas as alternativas acima

---

## Questão 9 (Múltipla Escolha)
No MVP, a View:

a) Tem lógica de negócio
b) Tem lógica de apresentação
c) Apenas exibe dados fornecidos pelo Presenter
d) Conhece Model diretamente

**Resposta:** c) Apenas exibe dados fornecidos pelo Presenter - View é passiva

---

## Questão 10 (Múltipla Escolha)
No MVP, o fluxo típico é:

a) Usuário → View → Presenter → Model → Presenter → View → Usuário
b) Usuário → Model → View
c) Usuário → View → Model
d) Não há fluxo

**Resposta:** a) Usuário → View → Presenter → Model → Presenter → View → Usuário
